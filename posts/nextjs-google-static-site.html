A new blog

December 25, 2023

I'm not sure which came first, the technology curiosity or the need for a blog, but that's often the case for me from an exploratory standpoint. I had been exploring modern development frameworks looking for a static site generator. I went to Jamstack and found that NextJS had nearly twice as much interest than any other. So I did a little research.

The most interesting part of this for me was prefetching. As defined by NextJS,

"Prefetching is a way to preload a route in the background before the user visits it."

And of course, upon futher research it appears to be a standard browser behavior that "developers can provide hints to the browser." - Mozilla web docs

So what's the difference?

According to a summary from ChatGPT the primary differnce is intelligent prefetching (it only prefetches links in the viewport), JavaScript prefetching, NextJS prefetches not only the HTML, but also the JS. NextJS also includes network considerations and idle time in prefetching. It also included reference to a great article from web.dev.

That's it for now. To get started yourself, check out the Create a Next.js App.
